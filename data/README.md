# data : Datasets

This folder contains all CSV datasets generated by the source scripts.
All data is synthetic : generated programmatically to simulate Zomato's
order and kitchen environment. No real Zomato data is used.

---

## Dataset Reference

### synthetic_orders.csv
The base dataset generated by `src/generate_dataset.py`.
Contains 5,000 simulated food delivery orders across 50 restaurants.

| Column | Description |
|---|---|
| order_id | Unique order identifier |
| restaurant_id | Unique restaurant identifier |
| restaurant_type | fast_food / casual_dining / cloud_kitchen |
| restaurant_size | small / medium / large |
| order_timestamp | Time the order was placed |
| for_timestamp | Time merchant tapped Food Order Ready in Mx app |
| rider_arrival_timestamp | Time rider arrived at restaurant (via GPS) |
| true_kpt_minutes | Actual kitchen prep time (ground truth) |
| recorded_kpt_minutes | KPT as recorded by FOR tap (may be biased) |
| concurrent_orders | Number of active orders at restaurant at that time |
| kitchen_load_score | Pre-computed load score (0-1) |
| mx_response_latency_sec | Time taken by merchant to respond in Mx app |
| rejection_flag | 1 if merchant rejected an incoming order, else 0 |
| is_peak_hour | 1 if order placed during lunch or dinner peak, else 0 |
| is_contaminated | Ground truth flag : 1 if FOR signal is rider-influenced |

---

### clean_orders.csv
Produced by `src/signal_validator.py`.
Contains only orders where the FOR signal passed the GPS delta validation check.
Rider-influenced FOR signals (delta under 60 seconds) have been removed.

Rows : 2,072 (out of 5,000 original orders)
Additional column added : `for_rider_delta_seconds`, `is_rider_influenced`

---

### enriched_orders.csv
Produced by `src/kitchen_load_score.py`.
Extends `clean_orders.csv` with computed Kitchen Load Score features per restaurant.

Additional columns added :

| Column | Description |
|---|---|
| order_hour | Hour of day extracted from order timestamp |
| order_date | Date extracted from order timestamp |
| rolling_avg_latency | Rolling average Mx response latency (3-order window) |
| rolling_rejection_rate | Rolling average rejection rate (3-order window) |
| rolling_concurrent | Rolling average concurrent orders (3-order window) |
| latency_score | Normalized latency component (0-1) |
| concurrent_score | Normalized concurrent orders component (0-1) |
| rejection_score | Normalized rejection rate component (0-1) |
| computed_kls | Final weighted Kitchen Load Score (0-1) |

---

### external_kitchen_rush.csv
Produced by `src/external_kitchen_rush.py`.
Contains hourly order volume simulations per restaurant broken down by source :
Zomato orders, dine-in orders, and competitor platform orders.

| Column | Description |
|---|---|
| restaurant_id | Unique restaurant identifier |
| restaurant_size | small / medium / large |
| hour | Hour of day (0 to 23) |
| is_peak | 1 if peak hour, else 0 |
| zomato_orders | Orders visible to Zomato |
| dine_in_orders | Dine-in orders invisible to Zomato |
| competitor_orders | Competitor platform orders invisible to Zomato |
| total_orders | Total kitchen load |
| true_kpt | KPT accounting for full kitchen load |
| zomato_only_kpt | KPT estimated using only Zomato orders |
| kpt_underestimation | Gap between true KPT and Zomato-only estimate |

---

### iot_events.csv
Produced by `src/iot_simulator.py`.
Contains 200 simulated IoT sensor events from an ESP32 device at the kitchen pass,
comparing IoT-verified food-ready timestamps against merchant FOR taps.

| Column | Description |
|---|---|
| order_id | Unique order identifier |
| order_timestamp | Time the order was placed |
| bag_ready_timestamp | Time IoT sensor detected bag placement |
| merchant_for_timestamp | Time merchant tapped Food Order Ready |
| true_kpt_minutes | Actual kitchen prep time (ground truth) |
| iot_kpt_minutes | KPT as recorded by IoT sensor |
| merchant_kpt_minutes | KPT as recorded by merchant FOR tap |
| is_contaminated | 1 if merchant FOR signal is rider-influenced |
| mqtt_payload | Full simulated MQTT event payload (JSON) |
